# Build Definition

## Introduction

Before a software application can be tested, it must first be compiled, packaged, and deployed into a testing environment. The software package prepared for testing is known as a **Build**.

Every time developers complete a set of features, fix bugs, or implement enhancements, they generate a new build and deliver it to the QA team. The Test Engineers then perform **Smoke Testing** to verify whether the build is stable before continuing with detailed testing.

Understanding the concept of a build is important because every phase of software testing begins with receiving a build from the development team.

---

## What is Build?

A **Build** is a software application that is **ready to go for testing**.

In simple terms, a build is a compiled version of the application that contains the latest source code, bug fixes, enhancements, and new features developed by the development team.

After a build is generated, it is deployed into the testing environment where the QA team begins testing.

---

## Build Generation

### Definition

**Build Generation** is the process of compiling the application's source code, integrating all software modules, resolving dependencies, and producing an executable version of the software.

Whenever developers complete a development task, a new build is generated.

### Build Generation Process

```text
Requirements
      │
      ▼
Design
      │
      ▼
Coding
      │
      ▼
Compile Source Code
      │
      ▼
Generate Build
```

---

## Build Deployment

### Definition

**Build Deployment** is the process of installing the generated build into the testing environment or production environment.

Once deployment is completed successfully, the QA team begins Smoke Testing.

### Build Deployment Workflow

```text
Development Team
        │
        ▼
Generate Build
        │
        ▼
Deploy to Testing Environment
        │
        ▼
Smoke Testing
        │
        ▼
Functional Testing
```

---

## Types of Build

Software builds are generally classified into the following types.

### 1. Development Build

- Generated by developers during software development.
- Used for internal development and debugging.
- May contain incomplete features.

---

### 2. Testing Build

- Delivered to the QA team.
- Used for Smoke Testing, Functional Testing, Regression Testing, and other testing activities.
- Should be reasonably stable before testing begins.

---

### 3. Release Build

- Final build approved after successful testing.
- Delivered to the customer or deployed into production.
- Considered stable and production-ready.

---

## Importance of Build

A software build plays a significant role in the Software Testing Life Cycle.

Its importance includes:

- **Provides the software version for testing.**
- **Allows QA Engineers to begin testing activities.**
- **Helps identify software defects.**
- **Supports continuous software development.**
- **Improves software quality through iterative testing.**
- **Enables version tracking and release management.**

---

## Real-Time Example

Suppose developers are working on an **Online Banking Application**.

After completing the Login Module and Dashboard Module, they generate **Build 1.0** and deploy it to the testing environment.

The QA team performs:

- Smoke Testing
- Functional Testing
- Integration Testing

Several defects are reported.

Developers fix those defects and generate **Build 1.1**.

The updated build is again deployed for testing.

This cycle continues until the software is stable and ready for production.

---

## Key Points

- **A Build is software that is ready for testing.**
- **Every build is generated after coding is completed.**
- **Builds are deployed into the testing environment.**
- **Smoke Testing is the first testing activity performed on every build.**
- **Multiple builds may be generated during software development.**
- **The final stable build becomes the Release Build.**

---

## Summary

A **Build** is the compiled version of a software application that is ready for testing. It is generated by the development team after integrating source code and implementing new features or bug fixes. After deployment, the QA team performs Smoke Testing to verify build stability before beginning detailed testing. Understanding build generation, deployment, and build types is essential for every Software Test Engineer because every testing activity starts with a software build.
