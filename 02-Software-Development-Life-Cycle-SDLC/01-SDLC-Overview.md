# Software Development Life Cycle (SDLC)

## Introduction

The **Software Development Life Cycle (SDLC)** is a systematic process followed to design, develop, test, deploy, and maintain high-quality software. It provides a structured approach to software development, ensuring that the final product meets customer requirements, is delivered within the estimated budget and timeline, and maintains high quality throughout its lifecycle.

SDLC is also referred to as the **Application Development Life Cycle (ADLC)**.

## Definition

**Software Development Life Cycle (SDLC)** is a systematic process for building software that ensures the **quality**, **correctness**, and **reliability** of the software product.

The primary objectives of SDLC are:

- **To develop high-quality software.**
- **To satisfy customer requirements and expectations.**
- **To complete the project within the predefined budget and timeline.**
- **To provide a structured approach for planning, designing, developing, testing, deploying, and maintaining software.**

Every phase of the SDLC produces specific deliverables that become the input for the next phase.

## Phases of SDLC

The Software Development Life Cycle consists of the following seven phases:

1. **Requirement Elicitation and Analysis**
2. **Feasibility Study**
3. **Design**
4. **Coding (Development)**
5. **Testing**
6. **Installation / Deployment**
7. **Maintenance**

## SDLC Workflow

```text
Requirement Analysis
        ↓
Feasibility Study
        ↓
Design
        ↓
Coding
        ↓
Testing
        ↓
Deployment
        ↓
Maintenance
```

## Phase 1: Requirement Elicitation and Analysis

### Description

This is the first and one of the most important phases of SDLC.

During this phase, the project requirements are collected from customers and stakeholders to clearly understand the expected functionality of the software.

The requirement gathering process is usually conducted by:

- **Business Analysts**
- **Project Managers**
- **Senior Developers**
- **Clients**
- **Domain Experts**
- **Stakeholders**

### Activities Performed

- **Collect business requirements.**
- **Analyze customer expectations.**
- **Identify project scope.**
- **Understand business opportunities and challenges.**
- **Identify project risks.**
- **Plan quality assurance activities.**
- **Prepare requirement documents.**

### Benefits

- **Provides a clear understanding of project requirements.**
- **Helps estimate project cost and timeline.**
- **Reduces misunderstandings during development.**

---

## Phase 2: Feasibility Study

### Description

After gathering the requirements, the next step is to determine whether the project is feasible.

The feasibility study is performed using the **Software Requirement Specification (SRS)** document, which contains all the functional and non-functional requirements of the software.

### Types of Feasibility

### Economic Feasibility

Determines whether the project can be completed within the allocated budget.

### Legal Feasibility

Checks whether the project complies with applicable laws, regulations, licenses, and cyber security policies.

### Operational Feasibility

Determines whether the proposed software can effectively satisfy the client's operational requirements.

### Technical Feasibility

Evaluates whether the existing hardware, software, and technical infrastructure can support the project.

### Schedule Feasibility

Determines whether the project can be completed within the specified deadline.

---

## Phase 3: Design

### Description

During the design phase, the software architecture and design documents are prepared based on the SRS document.

The design serves as the blueprint for software development.

There are two types of design documents.

### High-Level Design (HLD)

High-Level Design focuses on the overall system architecture.

It includes:

- **Module names**
- **Module functionality**
- **Relationships between modules**
- **Database tables**
- **Architecture diagrams**
- **Technology stack**

### Low-Level Design (LLD)

Low-Level Design provides detailed implementation information for every module.

It includes:

- **Module logic**
- **Database structure**
- **Input and output details**
- **Interface design**
- **Dependency handling**
- **Error messages**
- **Processing logic**

---

## Phase 4: Coding

### Description

After the design is completed, the development team begins writing the source code.

Developers implement the system using the selected programming language and development tools.

### Activities Performed

- **Develop software modules.**
- **Write clean and maintainable code.**
- **Follow coding standards and guidelines.**
- **Perform unit-level development.**
- **Use tools such as compilers, interpreters, and debuggers.**

### Key Point

The **Coding Phase** is generally the **longest phase** of the Software Development Life Cycle.

---

## Phase 5: Testing

### Description

Once development is completed, the software is moved to the testing environment.

The **Quality Assurance (QA)** team verifies whether the application satisfies the customer requirements.

### Activities Performed

- **Execute test cases.**
- **Identify software defects.**
- **Report bugs to developers.**
- **Verify bug fixes through re-testing.**
- **Perform regression testing whenever required.**

This process continues until the software becomes stable and satisfies the business requirements.

---

## Phase 6: Installation / Deployment

### Description

After successful testing, the software is released to the production environment.

The deployment team installs the application and verifies that it functions correctly in the customer's environment.

### Activities Performed

- **Release the software.**
- **Deploy the application.**
- **Verify deployment success.**
- **Resolve deployment-related issues if any.**

---

## Phase 7: Maintenance

### Description

Maintenance begins after the software is deployed and customers start using it.

The maintenance phase ensures that the software continues to function correctly throughout its lifecycle.

### Maintenance Activities

### Bug Fixing

Defects discovered after deployment are identified and corrected.

### Upgrade

The software is upgraded to newer versions with improved performance and compatibility.

### Enhancement

New features and functionalities are added based on customer requirements.

---

## Advantages of SDLC

- **Provides a systematic software development process.**
- **Improves software quality.**
- **Reduces development risks.**
- **Ensures proper documentation.**
- **Improves communication among teams.**
- **Helps complete projects within budget and schedule.**
- **Enhances customer satisfaction.**

## Real-Time Example

Consider the development of an **Online Shopping Application**.

- **Requirement Analysis:** Gather customer requirements such as user registration, product search, cart, and payment.
- **Feasibility Study:** Analyze budget, technology, and timeline.
- **Design:** Prepare application architecture and database design.
- **Coding:** Developers build each module.
- **Testing:** QA verifies all functionalities and reports defects.
- **Deployment:** Release the application to production.
- **Maintenance:** Fix bugs, improve performance, and introduce new features such as coupon systems and digital wallets.

## Key Points

- **SDLC stands for Software Development Life Cycle.**
- **It is also known as Application Development Life Cycle (ADLC).**
- **SDLC consists of seven sequential phases.**
- **Each phase produces deliverables for the next phase.**
- **SDLC helps deliver high-quality software within budget and schedule.**

## Summary

The **Software Development Life Cycle (SDLC)** is a structured framework used to develop high-quality software in a systematic manner. It consists of seven phases: Requirement Analysis, Feasibility Study, Design, Coding, Testing, Deployment, and Maintenance. Following SDLC helps organizations deliver reliable, maintainable, and customer-centric software while minimizing risks and ensuring efficient project management.
