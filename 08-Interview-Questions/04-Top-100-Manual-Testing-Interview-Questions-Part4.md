## Question 61

### What is Integration Testing?

Integration Testing is a level of testing in which two or more individually tested modules are combined and tested as a group to verify that they interact correctly.

### Objectives

- Verify module interaction
- Detect interface defects
- Validate data flow between modules

### Example

In an E-commerce application:

- Login Module
- Product Module
- Payment Module

These modules are integrated and tested together to ensure seamless functionality.

---

## Question 62

### What are the Different Types of Integration Testing?

The major integration testing approaches are:

- Big Bang Integration Testing
- Top-Down Integration Testing
- Bottom-Up Integration Testing
- Hybrid (Sandwich) Integration Testing

Each approach differs in the order in which modules are integrated.

---

## Question 63

### What is Big Bang Integration Testing?

Big Bang Integration Testing is an approach where all modules are integrated at once and tested together.

### Advantages

- Simple for small applications
- No need for stubs or drivers

### Disadvantages

- Difficult to isolate defects
- Time-consuming debugging
- Not suitable for large projects

---

## Question 64

### What is Top-Down Integration Testing?

Top-Down Integration Testing starts testing from the top-level modules and gradually integrates lower-level modules.

### Characteristics

- Uses **Stubs**
- High-level functionality is tested first

### Advantages

- Critical modules are tested early
- Early prototype availability

### Disadvantages

- Low-level modules are tested late
- Requires stubs

---

## Question 65

### What is Bottom-Up Integration Testing?

Bottom-Up Integration Testing begins with lower-level modules and gradually integrates higher-level modules.

### Characteristics

- Uses **Drivers**
- Utility modules are tested first

### Advantages

- Low-level defects detected early
- No need for stubs

### Disadvantages

- High-level logic tested late
- Requires drivers

---

## Question 66

### What is Hybrid (Sandwich) Integration Testing?

Hybrid Integration Testing combines both Top-Down and Bottom-Up approaches.

### Advantages

- Faster integration
- Better defect detection
- Suitable for large applications

### Disadvantages

- More complex to manage
- Requires experienced testers

---

## Question 67

### What is a Stub?

A Stub is a temporary program that simulates the behavior of lower-level modules that are not yet developed.

### Used In

- Top-Down Integration Testing

---

## Question 68

### What is a Driver?

A Driver is a temporary program that simulates higher-level modules that are not yet developed.

### Used In

- Bottom-Up Integration Testing

---

## Question 69

System Testing is the process of testing the complete integrated software system as a whole against the specified requirements.

### Objectives

- Validate end-to-end functionality
- Verify business requirements
- Ensure system readiness

Testing an entire banking application from login to money transfer and logout.

User Acceptance Testing (UAT) is the final phase of testing where the customer or end users verify that the software satisfies business requirements before deployment.

### Objectives

- Validate business needs
- Confirm software readiness
- Obtain customer approval
