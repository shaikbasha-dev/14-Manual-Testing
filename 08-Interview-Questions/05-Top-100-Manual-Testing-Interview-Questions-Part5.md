## Question 81

### What is Load Testing?

Load Testing is a type of Performance Testing that verifies how an application performs under the expected number of concurrent users or transactions.

### Objectives

- Measure response time
- Verify application stability
- Identify performance bottlenecks

### Example

Testing an e-commerce application with **5,000 concurrent users** during a sale.

---

## Question 82

### What is Stress Testing?

Stress Testing verifies the application's behavior beyond its expected capacity to determine the breaking point and recovery capability.

### Example

If a server is designed for **10,000 users**, testing with **25,000 users** helps determine how it behaves under extreme conditions.

---

## Question 83

### What is Spike Testing?

Spike Testing checks how the application responds to sudden and extreme increases or decreases in user load.

### Example

A ticket booking website experiences a sudden surge when booking opens for a popular event.

---

## Question 84

### What is Endurance (Soak) Testing?

Endurance Testing verifies that the application performs consistently over an extended period under a normal workload.

### Objectives

- Detect memory leaks
- Verify long-term stability
- Measure resource utilization

### Example

Running an application continuously for **72 hours**.

---

## Question 85

### What is Volume Testing?

Volume Testing evaluates how the application performs when processing a very large volume of data.

### Example

Testing a banking application with **10 million customer records**.

---

## Question 86

### What is Scalability Testing?

Scalability Testing determines whether the application can efficiently handle increasing workloads by adding resources.

### Example

Increasing server resources and verifying whether the application's performance improves proportionally.

## Question 92

### What is Defect Leakage?

Defect Leakage occurs when a defect is missed during testing and is discovered by the customer or end user after the software has been released.

### Causes

- Incomplete test coverage
- Missing test cases
- Time constraints
- Poor requirement analysis

## Question 93

### What is Defect Removal Efficiency (DRE)?

Defect Removal Efficiency (DRE) is a quality metric that measures how effectively defects are identified and removed before software release.

### Formula

DRE = (Defects Removed Before Release / Total Defects) × 100

### Benefits

- Measures testing effectiveness
- Improves quality processes
- Reduces production defects

## Question 94

### Explain a Typical Bug Life Cycle Scenario

A tester identifies a defect and reports it with complete details.

The defect progresses through the following stages:

1. New
2. Assigned
3. Open
4. Fixed
5. Retest
6. Verified
7. Closed

If the issue still exists after the fix, the defect is marked as **Reopened** and sent back to the developer.

## Question 99

### What is a Sprint and What Happens During a Daily Stand-up Meeting?

### Sprint

A Sprint is a fixed time-box (usually **2 to 4 weeks**) during which the team develops and delivers a potentially shippable product increment.
