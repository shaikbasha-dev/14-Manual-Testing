# Manual Testing Interview Questions and Answers

A comprehensive collection of the most frequently asked Manual Testing interview questions with detailed answers. This document is designed for students, freshers, QA Engineers, and professionals preparing for software testing interviews.

---

## Question 1

### What is Software Testing?

Software Testing is the process of evaluating and verifying that a software application works according to the specified requirements and is free from defects. It helps identify bugs, improves software quality, and ensures customer satisfaction before the product is released.

**Key Objectives**

- Identify defects
- Verify functionality
- Validate business requirements
- Improve software quality
- Reduce production risks

---

## Question 2

### Why is Software Testing Important?

Software testing is important because it ensures that the software functions correctly, meets customer expectations, and delivers a high-quality user experience.

**Benefits**

- Detects defects early
- Reduces development cost
- Improves product quality
- Increases customer confidence
- Prevents business losses
- Ensures security and reliability

---

## Question 3

### What is Manual Testing?

Manual Testing is a software testing process in which testers execute test cases manually without using automation tools. The tester verifies application functionality by interacting with the software like an end user.

**Characteristics**

- No automation tools required
- Human observation
- Suitable for exploratory testing
- Useful for UI and usability testing

---

## Question 4

### What is the Difference Between Manual Testing and Automation Testing?

| Manual Testing | Automation Testing |
|---------------|--------------------|
| Performed manually | Performed using tools |
| Slower execution | Faster execution |
| Suitable for exploratory testing | Suitable for regression testing |
| Less initial investment | Higher initial investment |
| Human intervention required | Minimal human intervention |

## Question 5

### What is Quality Assurance (QA)?

Quality Assurance (QA) is a process-oriented approach that focuses on improving development processes to prevent defects before they occur.

**QA Activities**

- Process improvement
- Standards implementation
- Audits
- Reviews
- Documentation

---

## Question 6

### What is Quality Control (QC)?

Quality Control (QC) is a product-oriented process that focuses on identifying defects in the developed software through testing.

**Examples**

- Executing test cases
- Reporting bugs
- Regression testing
- Functional testing

---

## Question 7

### What is the Difference Between QA and QC?

| QA | QC |
|----|----|
| Process-oriented | Product-oriented |
| Prevents defects | Detects defects |
| Proactive | Reactive |
| Focuses on process improvement | Focuses on product quality |

---

## Question 8

### What is Verification?

Verification is the process of checking whether the software is being developed according to the specified requirements.

**Question Answered**

> Are we building the product correctly?

**Examples**

- Reviews
- Walkthroughs
- Inspections
- Static analysis

---

## Question 9

### What is Validation?

Validation is the process of checking whether the developed software satisfies customer requirements.

**Question Answered**

> Are we building the right product?

**Examples**

- Functional Testing
- System Testing
- User Acceptance Testing

---

## Question 10

### What is the Difference Between Verification and Validation?

| Verification | Validation |
|-------------|------------|
| Checks documents | Checks software |
| Static process | Dynamic process |
| No code execution | Requires code execution |
| Done before validation | Done after verification |

**Phases**

2. Planning
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

## Question 15

### What is the Difference Between SDLC and STLC?

| SDLC | STLC |

## Question 16

### What is a Software Defect?

A software defect (bug) is a flaw or error in the software that causes it to produce incorrect or unexpected results.

**Example**

A login page accepts invalid credentials and still allows the user to log in.

---

## Question 17

### What is a Bug?

A bug is an error discovered during testing that causes software to behave differently from expected results.

**Example**

Clicking the "Submit" button crashes the application.

---

## Question 18

### What is an Error?

An Error is a mistake made by a developer while writing the source code.

**Example**

Using the wrong formula to calculate total price.

---

## Question 19

### What is a Failure?

Failure occurs when the software cannot perform the expected function during execution because of one or more defects.

**Example**

The application crashes when the user uploads a file.

---

## Question 20

### What is the Difference Between Error, Defect, Bug, and Failure?

| Term | Meaning |
|------|----------|
| Error | Human mistake made during coding |
| Defect | Problem introduced into the software because of an error |
| Bug | Defect identified during testing |
| Failure | Incorrect behavior observed while executing the software |

**Relationship**

Developer Error → Software Defect → Tester Finds Bug → User Experiences Failure
