# Software Testing Life Cycle (STLC)

## Introduction

Software Testing is not a single activity performed after software development. It is a systematic process consisting of multiple phases that help ensure the quality of a software application before it is delivered to customers.

The **Software Testing Life Cycle (STLC)** defines a sequence of activities performed by the Testing Team to plan, design, execute, monitor, and complete software testing efficiently.

Each phase in the STLC has its own objectives, deliverables, and entry and exit criteria, helping the testing process become organized, measurable, and effective.

---

## Definition

**Software Testing Life Cycle (STLC)** is a process or activity that we follow to test an application or software.

It consists of a series of well-defined phases that help Test Engineers perform software testing in a systematic manner to ensure that the software meets customer requirements and quality standards.

---

## Objectives of STLC

The main objectives of the Software Testing Life Cycle are:

- **Ensure software quality.**
- **Identify defects at an early stage.**
- **Verify business requirements.**
- **Validate software functionality.**
- **Reduce software defects before release.**
- **Deliver reliable software to customers.**

## Phases of Software Testing Life Cycle

The Software Testing Life Cycle consists of the following phases:

### 1. Requirement Study

The Testing Team studies the project requirements, understands the business functionality, identifies testable requirements, and clarifies any ambiguities before testing begins.

---

### 2. Test Planning

The Test Lead prepares the Test Plan, defines the testing strategy, allocates resources, estimates timelines, identifies risks, and plans the testing activities.

---

### 3. Test Case Design

The Test Engineers prepare detailed Test Cases based on the Software Requirement Specification (SRS) and business requirements.

Test data is also prepared during this phase.

---

### 4. Test Execution

The prepared Test Cases are executed on the software application.

The actual results are compared with the expected results.

If differences are found, defects are reported.

---

### 5. Requirement Traceability Matrix (RTM)

The Requirement Traceability Matrix ensures that every customer requirement has corresponding Test Cases and that no requirement is missed during testing.

---

### 6. Defect Tracking

All identified defects are logged, assigned to developers, monitored, and tracked until they are resolved and verified.

---

### 7. Test Execution Report

After testing is completed, the Testing Team prepares the Test Execution Report summarizing:

- Total Test Cases
- Passed Test Cases
- Failed Test Cases
- Blocked Test Cases
- Defects Found
- Overall Test Status

## Real-Time Example

Consider an **Online Banking Application**.

The Testing Team performs the following STLC activities:

- Study the Login, Fund Transfer, and Account Management requirements.
- Prepare the Test Plan.
- Design Login and Fund Transfer Test Cases.
- Execute the Test Cases.
- Report defects found during testing.
- Verify fixed defects.
- Prepare the Test Execution Report before software release.

This complete process represents the **Software Testing Life Cycle (STLC)**.
